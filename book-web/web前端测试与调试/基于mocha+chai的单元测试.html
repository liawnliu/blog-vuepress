# 使用 mocha+chai 进行单元测试

    web 前端使用JavaScript (opens new window)或TypeScript (opens new window)时开发，经常要对一些功能点进行测试，甚至还要提供直观的测试报告。一些测试技术用而生，例如jest (opens new window)和mocha (opens new window)等。这里我们选用mocha (opens new window)测试框架+chai (opens new window)断言库来进行测试，而测试报告使用mochawesome (opens new window)，覆盖报告使用nyc (opens new window)。

# mocha 前端测试框架

    mocha 是一个可以在node.js (opens new window)和浏览器上运行的前端测试框架，它支持不同风格的断言库，也支持同步异步测试、测试用例分组等功能。

    命令相关，一般是“mocha 文件名”，如果测试 ts 的话，需要在命令里加“ts-node/register”来编译运行。如果需要测试文件夹里的所有文件，那就用递归也就是加“-r 文件夹名”。

    代码示例：

import { expect } from "chai";
describe("Array", function () {
  describe("#indexOf()", function () {
    it("should return -1 when the value is not present", function () {
      expect([1, 2, 3].indexOf(4)).to.equal(-1);
    });
  });
});

# 使用 async/await

beforeEach(async function () {
  await db.clear();

  it("responds with matching records", async function () {
    const users = await db.find({ type: "User" });

# chai 断言集合库

    支持 expect、assert、should 断言语法

var assert = require("chai").assert,
  foo = "bar",
  beverages = { tea: ["chai", "matcha", "oolong"] };
assert.typeOf(foo, "string");
assert.typeOf(foo, "string", "foo is a string");
assert.equal(foo, "bar", "foo equal `bar`");
assert.lengthOf(foo, 3, "foo`s value has a length of 3");
assert.lengthOf(beverages.tea, 3, "beverages has 3 types of tea");

var expect = require("chai").expect,
  foo = "bar",
  beverages = { tea: ["chai", "matcha", "oolong"] };
expect(foo).to.be.a("string");
expect(foo).to.equal("bar");
expect(foo).to.have.lengthOf(3);
expect(beverages).to.have.property("tea").with.lengthOf(3);

var should = require("chai").should(),
  foo = "bar",
  beverages = { tea: ["chai", "matcha", "oolong"] };
foo.should.be.a("string");
foo.should.equal("bar");
foo.should.have.lengthOf(3);
beverages.should.have.property("tea").with.lengthOf(3);

# mochawesome 测试结果报告

    mochawesome 是一个用于 mocha 测试框架的自定义报告器，可以生成 html/css 报告在浏览器上查看，也可以本地查看。
    一般在命令里添加“--reporter mochawesome”就可使用 mochawesome。当然，最好清空之前的一些测试报告，那使用“rm -rf 文件夹名”。

# nyc 测试覆盖报告

    nyc 是一个测试覆盖率报告器，可以生成 html/css 报告在浏览器上查看，也可以本地查看。
    如果你用的 jest 其实就不用 nyc 了，已经自带了测试覆盖报告的类库。
    还要配置 nyc 覆盖范围，要配置 include 和 exclude 等信息。     给它确定报告生成的位置，使用“--report-dir 文件夹名”命令；要生成 html 形式的报告，使用“--reporter=lcov”命令；生成 text 形式的，使用“--reporter=text”，当然你两种都想要就两个命令都加上。

# 安装命令合集

  • npm install --save-dev mocha 用于局部安装 mocha
  • npm install --save-dev chai 用于局部安装 chai
  • npm install --save-dev mochawesome 用于局部安装 mochawesome
  • npm install --save-dev nyc 用于局部安装 nyc
  • npm install --save-dev @types/mocha 编写时要用到的库
  • npm install --save-dev @types/chai 编写时要用到的库

# 示例 package.json

{
  "name": "datastructures_ts",
  "version": "1.0.0",
  "description": "datastructures",
  "keywords": [],
  "author": "Liu",
  "license": "ISC",

    "clean": "rm -rf ./dist ./coverage ./.nyc_output ./coverage.lcov ./mochawesome-report",

    "test": "mocha -r ts-node/register ./test/ts/**/*.spec.ts ./test/ts/**/**/*.spec.ts --reporter mochawesome",
